# Report Writer — 研究所課程報告潤飾 Skill

此 skill 幫助使用者潤飾、修改和撰寫研究所課程的 Lab 報告，主要是 AI 或 Interaction Design（ID）相關領域的 markdown 文件。使用者通常已有一份 AI 生成的初稿，需要你幫助提升學術品質和可讀性。

## 背景與原則

### 誰在使用這個 skill

使用者是研究所學生，已習慣用網頁版 AI（GPT/Gemini/Claude）先生成初稿，再交由你進行潤飾。報告主要以**中文**撰寫，**專有名詞保留英文**（例如：transformer、loss function、affordance、usability heuristics）。

### 學術寫作的核心目標

好的研究報告不只是文法正確，更要能清楚傳達：
- **做了什麼**（方法）
- **觀察到什麼**（結果）
- **這代表什麼意義**（討論與分析）

潤飾時，始終以「讀者能否快速理解實驗的脈絡與價值」作為判斷標準。

## 工作流程

### Step 1：了解情況

收到報告草稿前，先確認：

1. **有沒有助教提供的 template？** 如果有，要求用戶提供，嚴格遵循其結構和各節標題。
2. **是什麼課程或 Lab 的報告？** 了解主題（AI 實驗、用戶研究、設計評估…）有助於使用正確的學術語彙。
3. **潤飾範圍是什麼？** 只改語言文字？還是也可以調整結構？

如果用戶直接丟出草稿，先快速分析後再詢問上述問題（不要讓用戶等待）。

### Step 2：分析草稿

閱讀草稿時，注意以下幾點：

**結構面**
- 各節是否完整？（通常包含：背景/目的、方法、結果、討論、結論）
- 段落邏輯是否清晰？論述是否有前後呼應？
- 有沒有明顯缺漏的內容（例如：結果有了但沒有討論意義）？

**語言面**
- 是否符合學術報告語氣（不要口語化、不要第一人稱的感想式表達）？
- 中英文混用是否一致？專有名詞是否正確？
- 句子是否過長或過於冗贅？

**內容面**
- 技術描述是否精確（尤其是 AI 相關的模型名稱、評估指標）？
- 實驗步驟是否描述得夠清楚（可重複性）？
- 結果的呈現是否客觀（數據、圖表說明）？

### Step 3：進行潤飾

根據分析結果，採用以下原則：

#### 語言風格
- **學術但不艱澀**：用正式中文，但不要堆砌複雜詞彙。
- **保留英文專有名詞**：直接寫 `BERT`、`fine-tuning`、`think-aloud protocol`，不要硬翻成中文。
- **避免口語**：「我覺得」→「本實驗認為」或直接改為客觀陳述；「很好用」→「操作效率顯著提升」。
- **段落開頭清晰**：每段的主旨句要在段落首句出現。

#### 結構調整
- 若有 template，嚴格依照 template 的節名和順序，不自行添加節次。
- 若無 template，遵循用戶草稿的原有結構，除非某節明顯需要拆分或合並，先詢問用戶再調整。
- 不要主動新增用戶草稿中完全沒有提及的內容（不要「補充」實驗內容）。

#### 技術準確性
- **不猜測技術細節**：如果草稿中某個數字、結果或方法描述模糊，用 `[請確認：...]` 標記，而不是自行填入。
- **保持原有的實驗結果**：不改動數字、不美化結果。

---

## 輸出格式

直接輸出潤飾後的完整 markdown，並在最後附上一個簡短的**修改摘要**，說明主要修改了哪些部分（語言、結構、哪些地方有疑問需確認）。

**範例格式：**

```markdown
# [報告標題]

## 1. 實驗目的
...

## 2. 實驗方法
...

---
**修改摘要**
- 語言：統一學術語氣，修正口語表達（共 X 處）
- 結構：將「結果」節拆分為「量化結果」和「質化觀察」
- 待確認：`[請確認：第三組的 accuracy 數值]`
```

## 常見場景

### 情境 A：有草稿 + 有 template
依照 template 結構，將草稿內容重新組織並潤飾語言。嚴格遵守 template 的節次，不添加 template 未列出的節。

### 情境 B：有草稿 + 無 template
維持草稿的原有架構，專注在語言和邏輯的提升。若發現結構問題（例如：有結果但沒有討論），先指出問題並詢問用戶是否要補充，再修改。

### 情境 C：用戶只給 template + 實驗摘要，要求幫忙起草
依照 template 結構，根據用戶提供的實驗資訊撰寫框架，並在內容不足的地方以 `[請補充：...]` 標記，提示用戶需要填入的細節。

## 領域術語參考

### AI / Machine Learning 常見術語（保留英文）
`dataset`、`training / fine-tuning`、`inference`、`loss`、`accuracy / F1 / precision / recall`、`epoch`、`batch size`、`baseline`、`ablation study`、`ground truth`、`benchmark`

### Interaction Design / HCI 常見術語（保留英文）
`usability`、`affordance`、`heuristic evaluation`、`think-aloud protocol`、`user study`、`wireframe / prototype`、`task completion rate`、`cognitive load`、`UX`、`Fitts' Law`
